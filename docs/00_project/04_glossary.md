# Glossary

| คำ | ความหมายในโครงการ Zayd |
|---|---|
| RAG | Retrieval-Augmented Generation: ค้นหลักฐานก่อนส่งบริบทให้โมเดลสร้างคำตอบ |
| Local-first RAG | ค้นฐานข้อมูลที่ self-host ก่อน แล้วค่อยใช้ provider ภายนอกเมื่อจำเป็น |
| Evidence | ข้อความหรือ metadata จากแหล่งข้อมูลที่ระบบอนุญาตให้ใช้สนับสนุนคำตอบ |
| Citation | ตัวระบุอ้างอิงที่ผูกกับเอกสาร เวอร์ชัน และ chunk ต้นทาง |
| Canonical reference | รูปแบบอ้างอิงมาตรฐาน เช่น ซูเราะฮ์:อายะฮ์ หรือชุดหะดีษ:เลขหะดีษ |
| Chunk | หน่วยข้อความที่ใช้ค้นคืน โดยต้องย้อนกลับไปยังเอกสารและตำแหน่งต้นฉบับได้ |
| Madhhab / มัซฮับ | สำนักนิติศาสตร์อิสลาม เช่น ชาฟิอี ฮะนะฟี มาลิกี ฮัมบะลี |
| Scholar | ผู้รู้ศาสนาที่ได้รับบทบาทตรวจหรืออนุมัติเนื้อหาในระบบ |
| Reviewer | ผู้ตรวจข้อความ คำแปล metadata หรือคำตอบที่ถูกรายงาน |
| Senior Scholar | ผู้มีสิทธิ์อนุมัติหรือระงับเนื้อหาศาสนาความเสี่ยงสูง |
| Abstain | ระบบงดตอบเมื่อหลักฐานไม่เพียงพอหรือคำถามต้องการผู้รู้เฉพาะกรณี |
| Evidence sufficiency | ผลประเมินว่าหลักฐานเพียงพอ บางส่วน ไม่เพียงพอ หรือขัดแย้ง |
| Provider | บริการภายนอกหรือภายใน เช่น LLM, embedding, knowledge API, vector store |
| Adapter | ชั้นเชื่อม provider ให้เป็น interface มาตรฐานของ Zayd |
| Source | แหล่งข้อมูลระดับเจ้าของ/โครงการ/หนังสือ/บริการที่เอกสารถูกนำมาใช้ |
| Source License | เงื่อนไขการเก็บ สร้าง embedding ใช้เชิงพาณิชย์ และแจกจ่ายข้อมูล |
| Prompt version | เวอร์ชัน prompt ที่ผ่านการอนุมัติและถูกบันทึกกับทุกคำตอบ |
| Policy version | เวอร์ชันกฎ เช่น risk, madhhab, safety และ license |
| Retrieval trace | ประวัติคำค้น filters ผลลัพธ์ คะแนน และ provider ของคำตอบหนึ่งรายการ |
| PWA | Progressive Web App ที่ติดตั้งบนมือถือได้จากเบราว์เซอร์ |
| MVP | Minimum Viable Product ที่มีวงจรหลักครบและพร้อมทดสอบกับกลุ่มปิด |
| Self-host | ติดตั้งระบบในโครงสร้างพื้นฐานของผู้ใช้หรือองค์กรเอง |
| Open weight | โมเดลที่แจกน้ำหนักให้ดาวน์โหลดและรันเองได้ ไม่เท่ากับ open-source เสมอไป |
| Regression test | Test ที่เพิ่มจากปัญหาจริงเพื่อป้องกันไม่ให้ปัญหาเดิมเกิดซ้ำ |
