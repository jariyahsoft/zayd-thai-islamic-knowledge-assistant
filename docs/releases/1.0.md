# Zayd 1.0 Release

## Release Overview

Zayd 1.0 is the first release of the Thai Islamic Knowledge Assistant — an AI-powered platform providing citation-backed answers to Islamic jurisprudence (fiqh), creed (aqidah), and daily-life questions in Thai, grounded in reliable scholarly sources.

## Release Scope

| Aspect | Detail |
|--------|--------|
| **Type** | Initial public release (closed pilot) |
| **Version** | 1.0.0 |
| **SBOM** | `docs/releases/sbom.md` |
| **Base image** | Python 3.12.3, Node 24.18.0 |
| **Infrastructure** | Docker Swarm, PostgreSQL with pgvector, Redis, S3-compatible storage |

## Architecture Summary

- **Monorepo** with Python services (uv workspace) and TypeScript frontend apps (pnpm workspace)
- **7 Python services**: common, api, orchestrator, retrieval, ingestion, worker, evaluation
- **3 Next.js apps**: web (user PWA), admin (governance console), reviewer (scholar workspace)
- **PostgreSQL + pgvector** for vector search, **Redis** for rate limiting and task queues
- **S3-compatible object storage** for document files

## Key Features

### User-facing
- Conversational chat interface with streaming answers
- Citation cards with source references (Quran, hadith, books)
- Madhhab-aware answers (Shafii default)
- Conversation history and saved answers
- User feedback submission

### Reviewer & Scholar
- Document review workspace with diff preview
- Scholar approval workflow with evidence requirements
- Feedback triage queue with root-cause classification
- Review queue with assignment and escalation

### Admin
- Provider and model management UI
- User and role management
- Source and license registry
- Evaluation run comparison dashboard
- Incident management

### AI Orchestration
- Question classification (intent, madhhab, risk detection)
- Risk policy engine (takfir, divorce, inheritance, health, violence)
- Retrieval-augmented generation with hybrid search
- Citation verification (existence, reference correctness, claim support)
- Abstention for insufficient/conflicting evidence

### Operations
- Structured JSON logging with secret redaction and request/trace ID propagation
- Prometheus-compatible metrics (latency, error rates, queue depth, cost)
- Grafana dashboard with API health, retrieval, citation, and provider panels
- Rate limiting, CORS, CSP, and SSRF protection

### Evaluation
- Reproducible benchmark runner with version-aware configurations
- Retrieval, citation, safety/abstention, and incident regression metrics
- Zayd-IslamicQA-TH starter dataset (public and private cases)
- SBOM generation and checksum verification

### Pilot Environment
- Invite-only registration with email hash allowlisting
- Isolated Docker Swarm overlay with separate secrets, storage, and volumes
- Pilot deployment validator script (secret namespace, dataset approval, checksum checks)

## Deployment

The system supports three deployment profiles:

1. **Minimal self-host** — single-machine Docker Compose with local LLM/embedding providers
2. **Development** — full Compose stack with postgres, redis, minio (see `infra/compose/development.yml`)
3. **Production** — Docker Swarm stack with external secrets and TLS (see `infra/compose/production.yml`)
4. **Pilot** — production overlay with invite-only access (see `infra/compose/pilot.yml`)

## Security

- All secrets are managed via Docker Swarm secrets or environment variables
- JWT-based authentication with optional MFA for privileged roles
- RBAC with separation of duties (uploader ≠ approver)
- Immutable audit logging for all mutations
- SSRF validation on provider URLs
- Prompt injection detection on user queries
- Automated penetration test suite (6 attack vectors verified)

## Known Limitations

- Guest sessions bypass some privacy protections (future work)
- Rate limiting is in-process (not Redis-backed) for MVP
- Thai and Arabic text normalization covers common patterns but not all Quranic orthographic variants
- Document parser supports PDF, DOCX, TXT, MD, HTML, JSON, CSV (not scanned images/OCR)
