# Software Bill of Materials (SBOM)

Zayd generates Software Bill of Materials (SBOM) for each release to provide
transparency into all dependency components, their versions, licenses, and
provenance. SBOMs support vulnerability management, license compliance, and
supply-chain security audits.

## SBOM Generation

The SBOM is generated by the `scripts/generate-sbom.py` script. It reads the
project's lock files (`uv.lock` for Python dependencies and `pnpm-lock.yaml`
for TypeScript/JavaScript dependencies) and produces a structured JSON document.

### Running Locally

```bash
# Generate SBOM to stdout
python3 scripts/generate-sbom.py

# Generate SBOM to a file
python3 scripts/generate-sbom.py --output release-sbom.json

# Verbose mode
python3 scripts/generate-sbom.py --verbose
```

### Requirements

- Python 3.12+ (standard library: `json`, `hashlib`, `tomllib`)
- PyYAML (for parsing `pnpm-lock.yaml`)
- The lock files must be present: `uv.lock` and `pnpm-lock.yaml`

## SBOM Format

The SBOM JSON document has the following structure:

```json
{
  "sbom_version": "zayd-sbom-v1",
  "metadata": {
    "generated_at": "",
    "generator": "zayd-generate-sbom",
    "git_commit": "<commit-sha>",
    "repo": "zayd-thai-islamic-knowledge-assistant"
  },
  "python": {
    "count": 52,
    "dependencies": [
      {
        "name": "fastapi",
        "version": "0.139.0",
        "type": "python",
        "source": "https://pypi.org/simple",
        "hashes": ["sha256:..."]
      }
    ]
  },
  "typescript": {
    "count": 293,
    "dependencies": [
      {
        "name": "react",
        "version": "19.2.17",
        "type": "javascript",
        "source": "npm",
        "integrity": "sha512-..."
      }
    ]
  },
  "checksums": {
    "uv.lock": "sha256-...",
    "pnpm-lock.yaml": "sha256-...",
    "pyproject.toml": "sha256-...",
    "package.json": "sha256-..."
  },
  "digest": {
    "algorithm": "sha256",
    "value": "sha256-of-the-entire-sbom"
  }
}
```

### Fields

| Field | Description |
|-------|-------------|
| `sbom_version` | Version of the SBOM specification |
| `metadata.generated_at` | ISO-8601 or SOURCE_DATE_EPOCH timestamp |
| `metadata.git_commit` | Git commit SHA of the repository at generation time |
| `metadata.generator` | Tool name that produced the SBOM |
| `python` | Python dependencies from `uv.lock` |
| `python.dependencies[].name` | Package name |
| `python.dependencies[].version` | Package version |
| `python.dependencies[].hashes` | SHA256 hashes of source and wheel artifacts |
| `typescript` | TypeScript/JavaScript dependencies from `pnpm-lock.yaml` |
| `typescript.dependencies[].name` | Package name |
| `typescript.dependencies[].version` | Package version |
| `typescript.dependencies[].integrity` | SRI integrity hash |
| `checksums` | SHA256 digests of key project files |
| `digest` | SHA256 digest of the entire SBOM document (excluding this field) |

## CI Integration

The SBOM is generated as part of the release workflow (`.github/workflows/ci.yml`).
It runs in the `license_check` job:

```yaml
- name: Generate SBOM
  run: python3 scripts/generate-sbom.py --output release-sbom.json
- uses: actions/upload-artifact@v4
  with:
    name: sbom
    path: release-sbom.json
```

## Docker Image Digests

For Docker images, the SBOM generator produces checksums for source
lockfiles rather than image digests. To generate Docker image SBOMs:

```bash
# Using syft (recommended)
syft zayd-api:latest -o spdx-json > docker-api-sbom.json

# Using docker sbom (Docker Desktop)
docker sbom zayd-api:latest > docker-api-sbom.txt
```

Docker image digests are computed by the registry at push time and recorded
in the release manifest:

```bash
docker image inspect zayd-api:latest --format '{{.Id}}'
```

## Secret Safety

The SBOM generator:

- **Never** includes environment variables, configuration values, or credentials
- **Never** reads `.env` files or secret files
- **Only** reads public lock files and manifest files
- Any file listed in `checksums` is a public manifest — never a secret file

To verify an SBOM contains no secrets:

```bash
python3 -c "
import json
with open('release-sbom.json') as f:
    data = json.load(f)
# Check no suspicious patterns
text = json.dumps(data)
for s in ['bot_token', 'api_key', 'password', 'AAGwa']:
    assert s not in text, f'Potential secret leak: {s}'
print('SBOM secret check passed')
"
```

## Reproducibility

The SBOM is deterministic for a given commit and lockfile state:

1. The same `uv.lock` and `pnpm-lock.yaml` produce identical dependency lists.
2. The `digest` field signs the document content and will be the same for
   identical inputs.
3. The `generated_at` field may vary if `SOURCE_DATE_EPOCH` is not set — set
   this environment variable to a fixed timestamp for fully reproducible output.

```bash
SOURCE_DATE_EPOCH=1741651200 python3 scripts/generate-sbom.py
```
