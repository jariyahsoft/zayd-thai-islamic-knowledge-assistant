# Security

## Reporting Security Issues

Do not file security vulnerabilities through public issues or pull requests.
Use the private security reporting path below:

- Security contact: `security@zayd.example`
- Alternate private path: maintainers' private contact channel

If a report must include logs, screenshots, or reproduction steps, redact:

- tokens, secrets, and credentials
- private user conversations
- personal data
- restricted datasets
- internal provider configuration

## Expectations

Security reports should describe:

- affected component
- impact
- steps to reproduce
- environment details
- whether the issue is reproducible with sanitized data

## Disclosure

Do not publicly disclose a vulnerability before remediation or an explicit coordinated disclosure decision.

## Response Boundaries

This project does not promise a fixed response time. Reports are triaged by maintainers and the security maintainer according to severity and available capacity.
